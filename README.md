# Story Time Universe — LG webOS TV

The native **LG webOS** viewer app for Story Time Universe. It talks to the same
production backend (`https://story-time.online`) as the [iOS](https://github.com/Dembula/Story-Time-Universe-iOS)
and [Android](https://github.com/Dembula/Story-Time-Universe-Android) clients,
and follows the identical viewer flow: **Sign in → Choose profile → Browse →
Watch**. Payments and account changes always open the TV browser — the app never
takes payments, exactly like the mobile clients.

Built with **React + TypeScript + Vite** and packaged as a webOS `web` app. webOS
runs web apps natively (HTML/CSS/JS on its Chromium-based WAM runtime), so this is
the platform's native application language.

## Highlights

- **Pure D-pad experience.** A geometric spatial-navigation engine (`src/tv/`)
  drives focus with the remote's arrow keys. The Magic Remote **pointer/cursor is
  disabled** so remotes that carry a pointer behave like a plain D-pad.
- **Best-in-class player** (`src/screens/Player.tsx`):
  - Title and back/exit controls **fade** in/out with activity.
  - A **clean timeline with no white scrubber knob** — just a gradient fill over a
    subtle track plus a faint buffered indicator.
  - HLS via `hls.js` (with native-HLS fallback for webOS), resume from last
    position, 10s skip with on-screen feedback, media-key support and an
    auto-playing **Up Next** for series.
- **Full-screen immersive Profiles page** — a wide backdrop **covers the entire
  1920×1080 canvas** and slowly cross-fades behind the profile picker.
- **Home that mirrors the production build** — featured hero carousel, Continue
  Watching, Trending, and all catalogue rows (Movies, Series, Animation, Comedy,
  Documentaries, …) that fill in as creators upload.
- **Smooth transitions** everywhere: focus scaling, page/row auto-scroll, and
  cross-fades.

## Viewer flow

1. **Sign in** with a viewer account (`credentials-viewer` via NextAuth).
2. Every launch lands on **Choose your profile** (never auto-restores a profile).
3. Select a profile (4-digit PIN supported via an on-screen pad).
4. **Home / Search / My List / Account** in a focusable left rail.
5. **Play** streams the HLS `playback-bundle`; **Trailer**, **credits/cast**,
   **episodes**, **more like this** and **behind the scenes** all supported.
6. **Renew / Change plan / Manage** deep-link out to the web app.

## APIs used (all from the production backend)

| Area        | Endpoints |
| ----------- | --------- |
| Auth        | `GET /api/auth/csrf`, `POST /api/auth/callback/credentials-viewer`, `GET /api/auth/session`, `POST /api/auth/signout` |
| Profiles    | `GET/POST /api/viewer/profiles`, `POST /api/viewer/profiles/active` |
| Catalogue   | `GET /api/content`, `GET /api/content/:id`, `GET /api/browse/search` |
| Playback    | `GET /api/content/:id/playback-bundle` (films + trailers), `GET/PUT /api/watch/progress`, `POST /api/watch`, `GET /api/watch/continue-watching` |
| Credits     | `GET /api/crew?contentId=`, `GET /api/people/:id/preview`, `GET /api/people/preview?crewMemberId=` |
| My List     | `GET/POST /api/watchlist` |
| Subscription| `GET /api/viewer/subscription` (read-only; checkout stays on web) |

## Develop

```bash
npm install
npm run dev        # http://localhost:5173 (test in a Chromium browser)
```

Because the browser enforces same-origin, live API calls during `npm run dev`
work best when the backend allows credentialed CORS for the dev origin. On the TV
itself the packaged app makes credentialed requests to `story-time.online` like
the mobile clients do.

## Build & package for webOS

```bash
npm run build      # outputs ./dist (includes appinfo.json + icons)

# With the LG webOS TV CLI (ares) installed and a TV/emulator registered:
npm run package        # ares-package dist -o ./bin  -> ./bin/com.storytime.universe_1.0.0_all.ipk
npm run install-app    # ares-install ./bin/*.ipk
npm run launch         # ares-launch com.storytime.universe
```

Install the CLI from the [webOS TV SDK](https://webostv.developer.lge.com/develop/tools/cli-installation).

## Remote key map

Arrows move focus · **OK** activates / play-pause · **Back** goes back (exits at
root) · Play/Pause/Stop/FF/Rew media keys are handled in the player. In the
player, Left/Right skip 10s.

## Notes

- Written for webOS 4.x+ (JS is transpiled to ES2018 for older TV Chromium).
- The Magic Remote pointer is hidden globally (`cursor: none` + `cursorStateChange`).
- Icons are placeholders generated by `node scripts/gen-icons.mjs`; replace with
  final brand art before store submission.
